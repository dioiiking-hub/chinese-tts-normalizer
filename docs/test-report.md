# chinese-tts-normalizer 测试报告

## 测试背景

在数字人/电商直播场景中，TTS 引擎需要准确播报产品型号、技术参数和价格。
我们使用同一段标准文本，对 5 款主流中文 TTS 引擎进行横向测试。

## 测试文本

```
搭载R7-5800H、i7-7700和i7-11800H的联想R9000P和微星GP76同时搭载RTX5090Ti最强显卡，
搭载麒麟9000处理器和天玑9400芯片的掌机支持4K和1080P输出，
当年的麒麟980处理器玩720P游戏帧率爆表，搭载5000mAh电池续航也能有20小时。
华为致敬mate8的经典作品mate10是一代经典，而今mate80的发布，
这让我想到了苹果iPhoneX，近期荣耀500系列。
```

### 各平台错误详情

**火山引擎 豆包TTS2.0：**
- i7-11800H：1读成"一"（正确是"幺"）
- 联想R9000P：9000读成"九零零零"（正确是"九千"）
- mate80：80读成"八十"（正确是"八零"）
- iPhoneX：X读成英文X（应读"十"）

**MiniMax speech-2.8-hd：**
- 1080P：1080读成"一千零八十"（正确是"幺零八零"）
- 5000mAh：读成"五零零零ma"（应是"五千毫安时"）
- RTX5090Ti：Ti读成"ti"（正确是"钛"）

**阿里 Qwen3-tts-instruct：**
- i7-11800H：11800读成"一万一千八百"（正确是"幺幺八零零"）
- 联想R9000P：9000读成"九零零零"（正确是"九千"）

**讯飞 超拟人语音合成：**
- R7-5800H：5800H读成"五千八百小时"（正确是"五八零零H"）
- i7-11800H：11800读成"一万一千八百小时"

**百度 大模型语音合成：**
- 联想R9000P：全平台唯一共同错误
- mate80、iPhoneX：读法错误

## 结论

**没有一家全对。** 联想R9000P（全军覆没）、iPhoneX（仅1家正确）、mate80（仅1家正确）是最大痛点。
